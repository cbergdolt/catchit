Development Plan

Relational Schema

  • GeneralJobPosting(JobID, JobTitle, Location, Deadline, URL, Source, Other Details)
    • FDs: {JobID > URL}
  • UserJobOfInterest(UserID, JobID, Stage)
  • Notes(UserID, JobID, NoteID, Content)
    • FDs: {UserID,JobID,NoteID > Content}
  • User(UserID, Email, Name, Password)
    • FDs: {Email > Password}
  • ExpressedInterest(UserID, JobID)
  • Interview(UserID, JobID, CorrespondenceID, Date, Location)
  • Email(UserID, JobID, CorrespondenceID, Recipient, Message)
  • NetworkingEvent(UserID, JobID, CorrespondenceID, EventName, Person, Relationship)

Note: Each relation's keys are underlined.

Databases/Software Platforms

SQL, php

Where/How we will get data

LinkedIn, Indeed, other job postings. The information will automatically be parsed, if possible. Otherwise, it is up to the user to input the information from the website into the database tool.

Division of Labor

While we expect the division of labor to change on a weekly basis depending on each member's schedule, the general division of labor will be as follows:

  • Data entry - Jamie, Erin
  • Data retrieval/displaying - Carolyn, Troy

We expect these roles to be flexible, and also anticipate that we will spend equal amounts of time working on the advanced functionalities.
